    #git

    获取 Git 仓库

    • 初始化仓库 git init
    • 克隆现有的仓库 git clone

    跟踪文件的修改

    • git add filename 增加跟踪文件或将已跟踪文件添加到暂存区

    • git rm filename 删除跟踪文件

      • 默认删除文件跟踪状态和本地文件
      • –cache参数 只删除跟踪状态而不删除本地文件，之后在.gitignore修改即可不再提示修改该文件

    • git mv sourcefile targetfile 跟踪文件改名

    • .gitignore 忽略某文件的跟踪

    • git ls-files 列出所有跟踪文件

    对工作区的修改

    git checkout -- filename 撤销某文件的修改到上次提交

    对暂存区的修改

    • git add 将已跟踪文件添加到暂存区
    • reset Head 将文件从暂存区删除

    git状态检查和历史

    • 检查当前文件状态 git status
    • 查看文件修改 git diff
      • 默认情况下比较工作区文件（未暂存）和暂存区文件
      • –cached 查看暂存区和上次提交的区别
    • 查看提交历史 git log
      • 默认会列出每次提交的SHA-1校验和，作者名字和邮箱，电子邮件地址，提交时间，提交说明
      • –stat 额外显示出每次提交差异的简略说明
      • –graph 图形化显示分支，合并历史
      • –pretty 自定义显示，与graph联合使用，如： git log --pretty=format:"%h %s" --graph

    扩展、标记和调校历史记录

    • 提交更新

      • git commit
        • 默认情况下提交修改
        • -a 跳过提交暂存区而直接将已修改的跟踪文件提交
        • –amend 重新提交上次修改
      • git checkout -- filename 撤销某文件的修改到上次提交

    • 撤销操作

      • git commit --amend 重新提交
      • git reset HEAD <file> 取消暂存
      • git checkout -- <file> 撤消对文件的修改- 将它还原成上次提交时的样子

    • 打标签

      • 列出标签
        • git tag 列出标签
        • git show 看到标签信息与对应的提交信息
      • 创建标签
        • 附注标签（annotated） git tag -a <tagname> 包含打标签者的名字、电子邮件地址、日期时间；还有一个标签信息
        • 轻量标签 没有保存任何其他信息 git tag <tagname>
        • 后期打标签 git tag -a <tag> <部分校验和>
      • 共享标签
        • git push origin [tagname]
        • git push origin –tags 把所有不在远程仓库服务器上的标签全部传送到那里

    远程仓库

    • 查看远程仓库 git remote
      • 默认列出指定的远程仓库的简写
      • git remote -v列出读写所需的git仓库简写及对应URL
      • 查看远程仓库 git remote show origin
    • 添加远程仓库 git remote add <shotname> <URL>
      • 添加新的远程 Git 仓库，同时指定一个你可以轻松引用的简写：
      • 之后可以使用git fetch shotname获取远程仓库
    • 从远程仓库拉取
      • git fetch <shotname> 访问远程仓库，拉取所有分支的引用，可以随时查看和合并，但不会自动合并
      • git pull
    • 远程仓库的移除与重命名
      • git remote rename old new
      • git remote rm shotname

    分支

    • 本地分支

      • 分支创建

        • git branch <branchname> 仅仅 创建 一个新分支，并不会自动切换到新分支
        • git checkout -b <branchname> 新建一个分支并同时切换到那个分支上

      • 分支切换 git checkout <branchname> 更改HEAD指针的指向,并将工作目录恢复成该分支所指向的快照内容

      • 分支查看

        • git log --all --oneline --decorate --graph 查看所有分支提交
        • git branch 查看分支

      • 分支合并 git merge

        • Fast-forward master 分支所指向的提交是你当前提交的直接上游， Git 只是简单的将指针向前移动
        • 分支冲突 用编辑其修改冲突文件并git add手动加入暂存区

      • 删除分支

        • git branch -d <branchname>
        • git branch -D <branchname> 强制删除分支，放弃合并

    • 远程分支

      • 远程分支创建

        • git remote add <shotname> <URL>
        • git checkout -b <local_branch> origin/<remote_branch>

      • 远程分支查看 git remote

      • 远程分支获取

        • git fetch origin(remote_name)
        • git pull origin(remote_name) master(branch_name)

      • 远程分支推送 git push origin(remote_name) local_branch:remote_branch

      • 远程分支删除 git push origin --delete serverfix

    • 变基

      • 将提交到某一分支上的所有修改都移至另一分支上
      • git checkout experiment
      • git rebase master

    • 回到 master 分支，进行一次快进合并

    • git checkout master

    • git merge experiment

    应用

    本地项目git初始化并提交远程仓库

    1. 先在远程仓库(如github)创建项目,为了避免错误,不要初始化 README, license, 或者gitignore文件

    2. 切换到你的本地项目目录,初始化本地仓库 git init 或 git clone

    3. 添加文件到本地仓库 git add .

    4. 提交文件 git commit -m “First commit”

    5. 添加远程仓库地址到本地仓库(若为clone则不需要)

      git remote add origin {远程仓库地址} //设置远程仓库
      git remote -v

    6. 为推送当前分支并建立与远程上游的跟踪

      git push –set-upstream origin master
      若出现拒绝合并无关历史(在远程库中创建README等)则运行先运行git pull origin master –allow-unrelated-histories

    给别人贡献代码

    准备工作

    1. 首先通过 git clone 将项目克隆到本地（我早已拉下来，跳过此步骤）
    2. git pull 拉取最新代码（将所有的change都同步到本地）
    3. 将 原项目 fork 到 自己的github上,并复制代码url
    4. 在本地添加第二个仓库地址：git remote add [nickname] [your url]

    修改

    1. 修改bug 或 新增功能
    2. git commit [file1] [file2] ... -m [message] 本地提交代码

    同步到github中并发到原项目

    1. git push [nickname] 将代码 push 到自己的项目里，nickname就是添加的第二个仓库的名字
    2. 自己项目内，点击 pull requests -》 new pull request 将本次修改提交到原项目进行同步。
