    使用fabric2实现网站自动部署

    运行远程命令

    fabric通过SSH在远程系统上执行shell命令.默认情况下，远程程序的输出将直接打印到终端并进行捕获。

    from fabric import Config, Connection
# 配置sudo命令的密码
config = Config(overrides={'sudo': {'password': 'password_root'}})
# 以明文方式配置用户登录密码
conn = Connection('ip', user="name", config=config, connect_kwargs={"password": "passwd"})
conn.run("whoami") # 返回ayuan
conn.sudo("whoami") # 返回root
conn.close()

    关于Connect类的详情参阅Connection
    run, sudo, local(运行本地命令) 都是调用了invoke.run, 函数详情如invoke.run

    传输文件

    可以通过来传输文件.
    其参数行为类似于cp和scp/sftp

    result = conn.put("fabfile.py", "/home/ubuntu/fabfile_put.py")
print("Uploaded {0.local} to {0.remote}".format(result))

result = conn.get("/home/ubuntu/fabfile_put.py", "fabfile_get.py")
print("Download {0.remote} to {0.local}".format(result))

    返回
    Uploaded /home/…/fabfile.py to /home/…/fabfile_put.py
    Download /home/…/fabfile_put.py to /home/…/fabfile_get.py

    详情参见Connection.put和Connection.get

    多服务器连接

    from fabric import SerialGroup as Group
results = Group('web1', 'web2', 'mac1').run('uname -s')
for connection, result in results.items():
    print("{0.host}: {1.stdout}".format(connection, result))

    Group返回GroupResultdict-like对象

    fab命令行工具

    在fabfile.py中打包命令并用@task装饰器暴露函数

    from fabric import task
@task
def upload_and_unpack(c):
    if c.run('test -f /opt/mydata/myfile', warn=True).failed:
        c.put('myfiles.tgz', '/opt/mydata')
        c.run('tar -C /opt/mydata -xzvf /opt/mydata/myfiles.tgz')

    在命令行中

    $ fab --list
Available tasks:

  upload_and_unpack

$ fab -H web1,web2,web3 upload_and_unpack

    常用方法详解

    Connection

    class fabric.connection.Connection(host, user=None, port=None, config=None, gateway=None, forward_agent=None, connect_timeout=None, connect_kwargs=None, inline_ssh_env=None)

    此类继承自Invoke的Context，因为它是命令，任务等可以在其中运行的上下文。
    它还封装了Paramiko SSHClient实例，使用从中生成的SSHClient和Channel实例执行有用的高级操作。

    许多SSH特定选项 - 例如指定私钥和密码，超时，禁用SSH代理等 - 由Paramiko直接处理，应通过构造函数的connect_kwargs参数指定。

    参数
    class fabric.connection.Connection()
    __init__(host, user=None, port=None, config=None, gateway=None, forward_agent=None, connect_timeout=None, connect_kwargs=None, inline_ssh_env=None)
    • host 主机名或ip地址
    • user 登录用户名
    • port (int) 端口
    • config 可以使Config或fabric.config.Config对象或invoke.config.Config对象
    • gateway 用作此连接的代理或网关的对象。 可以接受以下两种参数

      另一个Connection（用于ProxyJump样式网关）;

      shell命令字符串（用于ProxyCommand样式样式网关）。

    默认值：无，表示不会发生网关（除非另行配置;如果想要在运行时覆盖已配置的网关，请指定gateway = False。）

    • forward_agent（bool） 是否启用SSH代理转发。 默认值：config.forward_agent。
    • connect_timeout（int） - 连接超时，以秒为单位。 默认值：config.timeouts.connect。
    • connect_kwargs（字典） - 关键字参数逐字传递给SSHClient.connect（调用open时）。
      可以接受多种验证方式

      私钥文件 connect_kwargs={“key_filename”:”file_path”}

      加密的私钥文件 配置connect_kwargs.passphrase或者使用fab命令时加入参数–prompt-fro-passphrase以命令行交互式方式写入密码

      秘钥对象 详情参阅API, 将该对象配置为connect_kwargs.pkey

      SSH 客户端 此为默认情况,会连接到正在运行的SSH代理(在Unix上为SSU_AUTH_SOCK, 在Window上为Pageant),但是请注意,在我的ubuntu电脑上出错, 可以使用connect_kwargs.allow_agent=False禁用,详情参阅遇见过的坑

      明文密码 通过connect_kwargs.password方式

      GSSAPI 详情参阅文档

    • inline_ssh_env（bool） -
      是否将“inline”环境变量作为命令字符串前面的前缀发送（export VARNAME = value && mycommand），而不是尝试通过SSH协议本身提交它们（这是默认行为）。 如果远程服务器具有受限制的AcceptEnv设置（这是常见的默认设置），则必须执行此操作。
    返回值

    Connection对象的方法（如run）通常返回invoke.runners.Result（或其子类）的实例。

    • stdout（str） - 子进程的标准输出。
    • stderr（str） - 与stdout相同但包含标准错误（除非通过pty调用进程，在这种情况下它将为空）
    • encoding（str） - 本地shell环境使用的字符串编码。
    • command（str） - 执行的命令。
    • shell（str） - 用于执行的shell二进制文件。
    • env（dict） - 用于执行的shell环境。 （默认为空字典）
    • exited（int） - 表示子进程’退出/返回代码的整数。
    • pty（bool） - 描述子进程是否使用pty调用的布尔值;见Runner.run。
    • hide（元组） - 流名称的元组（无，一个或两个（’stdout’，’stderr’））在执行生成命令时对用户隐藏;这是从Runner.run的hide参数派生的标准化值。
    >>> result.stdout.strip() == 'Linux'
True
>>> result.exited
0
>>> result.ok
True
>>> result.command
'uname -s'
>>> result.connection
<Connection host=web1>
>>> result.connection.host
'web1'
    Connection.get(remote, local=None, preserve_mode=True)
    * remote (str) – 要下载的远程文件。 **Note** 不解析~, 默认工作目录为用户家目录 * local – 存储下载文件的本地路径，或类文件对象。 * preserve_mode（bool） - 是否对os.chmod本地文件进行匹配，使其与远程文件的模式匹配（默认值：True）。

    返回Result对象

    Connection.put(local, remote=None, preserve_mode=True)
    * remote (str) – 要下载的远程文件。 **Note** 不解析~, 默认工作目录为用户家目录 * local – 存储下载文件的本地路径，或类文件对象。 * preserve_mode（bool） - 是否对os.chmod本地文件进行匹配，使其与远程文件的模式匹配（默认值：True）。

    返回Result对象

    invoke.run(command, **kwargs)

    * command（str） - 要执行的shell命令。 * bool(布尔) - 当执行的命令以非零状态退出时，是否警告并继续，而不是引发UnexpectedExit。 默认值：False。 * hide - 允许调用者禁用run的将子进程'stdout和stderr复制到控制终端的默认行为。 指定hide ='out'（或'stdout'）以仅隐藏stdout流，hide ='err'（或'stderr'）仅隐藏stderr，或hide ='both'（或True）以隐藏两个流。 * pty（bool） - 默认情况下，run直接连接到调用的进程并读取其stdout / stderr流。。 要使用pty(伪终端)而不是默认行为，请指定pty = True。 * fallback (bool) – 控制自动回退功能 * echo（bool） - 控制run是否在执行命令之前将命令字符串打印到本地stdout。 默认值：False。 * env（dict） - 默认情况下，子进程接收Invoke自己的环境（即os.environ）的副本。在这里提供dict来更新子环境。例如`run('command', env={'PYTHONPATH': '/some/virtual/env/maybe'})`会修改PYTHONPATH环境变量 * replace_env（bool） - 当为True时，使子进程接收给予env的字典作为其整个shell环境，而不是更新os.environ的副本。 默认值：False。 * encoding（str） - 覆盖自动检测子进程对其stdout / stderr流使用哪种编码（默认为default_encoding的返回值）。 * out_stream - 应该写入子进程'标准输出的类文件流对象。 如果为None（默认值），将使用sys.stdout。 * err_stream - 与out_stream相同，除标准错误外，默认为sys.stderr。 * watchers - StreamWatcher实例列表，用于扫描程序的stdout或stderr，并可以响应模式或其他启发式方法写入其stdin * echo_stdin（bool） - 是否将数据从in_stream写回out_stream。

    遇见过的坑

    运行时报错如下 
    ayuan@ayuan-ubuntu:~/code/python/webapp/day15$ python fabfile.py 
Exception: key cannot be used for signing
Traceback (most recent call last):
  File "/home/ayuan/.local/lib/python3.6/site-packages/paramiko/transport.py", line 2036, in run
    handler(self.auth_handler, m)
  File "/home/ayuan/.local/lib/python3.6/site-packages/paramiko/auth_handler.py", line 298, in _parse_service_accept
    sig = self.private_key.sign_ssh_data(blob)
  File "/home/ayuan/.local/lib/python3.6/site-packages/paramiko/agent.py", line 418, in sign_ssh_data
    raise SSHException("key cannot be used for signing")
paramiko.ssh_exception.SSHException: key cannot be used for signing

    问题详情可见
    question/183769以及
    bug/912123

    总结来说就是环境变量SSH_AUTH_SOCK导致的坑，可以设置connect_kwargs.allow_agent=False禁用SSH agent方式,或只要是没有该变量的用户，如root即可, sudo python fabfile.py可正常运行
