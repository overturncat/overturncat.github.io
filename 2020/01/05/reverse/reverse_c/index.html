    裸函数

    在空函数测试中，虽然我们并没有用到函数栈，但是编译器都默认分配了64字节的内存，并初始化为0XCCCCCCCC，并且有传参、保存现场、堆栈提升、恢复现场、堆栈恢复等操作。
    这一切都是编译器做的。而有时我们需要自己按照自己的意愿设计函数结构，而不是让编译器把一个空函数都转成十几行代码。那我们就需要自己实现了。比如，empty空函数在调用后由于是空函数，在jmp到的地址只要一行ret就可以达到相同的功能。如下所示:

    __declspec(naked)即用来修饰函数，被修饰的函数即称为裸函数，裸函数中如果不进行任何操作，则不能被直接调用，因为裸函数是不被编译器处理的，即不会生成基本的结构，而call调用时修改了eip，跳转过去后只有int 3中断指令，是不能跳转回来的，所以我们应该用__asm{}格式来编写汇编代码，在其中只写一条ret即可实现最简单的函数调用

    那么我们可以使用这种裸函数的方式进行比较自由的函数操作，而不必让编译器替我们做主，当然我们也可以在__asm{}中模拟编译器的做法，如下所示：

    调用约定

    调用约定 传参方式与堆栈平衡处理方式
    __cdecl(默认) 从右至左入栈,主调函数做堆栈平衡:外平栈(ret回去后add esp,x)
    __stdcall 从右至左入栈,被调用函数做堆栈平衡:内平栈(ret x)
    __fastcall ECX/EDX接收最左边两个,其它参数同上,被调者做堆栈平衡:内平栈(ret x)


    主函数入口

    1. main 或WinMain 是“语法规定的用户入口”，而不是“应用程序入口”。应用程序入口通常是启动函数。
    2. mainCRTStartup 和 wmainCRTStartup 是控制台环境下多字节编码和Unicode 编码的启动函数.
      (即KERNEL->mainCRTStartup->main)
      而WinMainCRTStartup 和wWinMainCRTStartup 是windows 环境下多字节编码和Unicode 编码的启动函数.

    C语言流程

    IF

    • if…

    • if…else…

    • if…elseif…else…

    switch

    • case数量较少时, 会生成类似于if…else…的结构

    • 当case较多且连续时, switch 大表生成

    1. 分支少于4的时候,switch分汇编会生成类似于if…elseif的代码
    2. case之后常量的顺序并不影响大表的生成
    3. 反汇编时大表的结构与最大值与最小值的差值有关,
      case101: …case102:…case103:…case104:…会生成类似代码, 唯一的不同为sub ecx, 65H
    4. 若所有数连续相近, 即使中间少了几个点(如1,4,5,7,8),依然生成大表,且表中缺失点处填充default地址
      若不连续相近(如3, 101,102,103),则生成类似于if…elseif…的代码
    • 当case较多且连续, 但中断的点较多时, 小表的生成

    当中间的值缺少到一定程度就会生成小表,小表紧挨着大表,起着重定向的作用

    • 将case后面常量表达式改成毫不连续的值，二叉树排序
