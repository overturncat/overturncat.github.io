    OllyDBG

    一般原理

    • 在调试期间的断点［breakpoints］、标签［labels］、注释［comment］信息会保存在扩展名为.UDD的文件中, 当OllyDbg下一次加载该模块时，它会自动恢复所有的调试信息(默认名是 程序名.udd, 忽略路径, 后缀, 时间戳, RCR信息, 在选项–>调试设置–>安全 中修改)
    • dll文件的相关信息也会保存到.UDD文件中, 即使是不同的程序调用

    选项

    界面选项

    外观配置, 无需更改

    调试选项

    安全

    忽略路径和扩展名［Ignore path and extention］
    OllyDbg 将用户自定义的数据，如标签、注释、断点等保存到.udd文件中。通常这个文件会以可执行模块的名字命名，并且保存到 OllyDbg 的目录中。然而，可执行文件和DLL文件是可以具有相同名字的，不同目录中的DLL文件也可以具有相同名字。为了避免冲突，OllyDbg 1.09 在文件名中添加了后缀（_1,_2等）。这种做法的负面影响是如果您把被调试程序移动到其他文件夹时，以前的标签和断点将无法使用。选项“忽略路径和扩展名”［Ignore path and extention］在检查数据正确性时，会忽略 .udd 文件中的文件名后缀、路径和扩展名。

    忽略时间戳［Ignore timestamp］
    忽略时间的不同

    忽略代码段CRC检验［Ignore CRC of code section］
    当应用程序加载新模块时，OllyDbg会在在恢复用户自定义的数据之前，先要将数据的最近改动、模块长度、代码段CRC校验，与文件中列出的对应内容进行比较。如果您有几个相同名字的DLL或相同程序的几个不同版本，这将是最低的安全限度，保证不会识别错误。您可以禁止CRC和时间戳比较。如果您修改了可执行模块，最后一个选项将特别有用。

    其他 保持默认

    调试

    保持默认即可

    事件

    选择在发生相应的事件时自动中断
    第一次暂停在［Make first pause at］：

    • 系统断点［System breakpoint］
    • 主模块入口［Entry point of main module］
    • WinMain（函数如果位置已知）［WinMain (if location is known)］

    当新模块（动态链接库）加载时中断［Break on new module (DLL)］
    如果该选项开启，则当被调试程序加载新模块时，OllyDbg将停止执行。不同于一些其他的调试器，OllyDbg将一直等到模块被完全加载到内存才停止，所以您可以立即对这模块浏览，设置断点等操作。

    当模块（动态链接库）卸载时中断［Break on module (DLL) unloading］ .
    如果该选项被激活，OllyDbg将在操作系统卸载某些模块时暂停被调试程序。注意，此时模块已经不可浏览。

    当新线程运行时中断［Break on new thread］
    如果想让OllyDbg在模块启动新线程时暂停执行，请开启该项。

    当线程中止时中断［Break on thread end］
    如果该选项被激活，OllyDbg将在某些线程结束时停止被调试程序。

    当调试字符串时中断［Break on debug string］
    如果想让OllyDbg在每次被调试程序发出调试字符串（通过调用OutputDebugString）时暂停，请选中该项。

    代码跟踪

    Hit跟踪
    选择部分代码跟踪, 未执行代码显紫色, 已执行代码显红色.当退出Hit跟踪的时候，Run跟踪也会同时退出。

    功能描述 功能
    对选择的代码部分进行Hit跟踪 右键–>Hit跟踪–>添加选择部分
    对当前函数过程进行Hit跟踪。 右键–>Hit跟踪–>添加函数过程
    从选择部分中移除Hit跟踪和Run跟踪 右键–>Hit跟踪–>从选择的部分移除
    从当前模块的代码段中移除所有Hit跟踪和Run跟踪 右键–>Hit跟踪–>从选择的模块中移除
    选择代码均不进行Hit跟踪。 右键–>Hit跟踪–>标记选择部分不跟踪
    当前模块命令均不进行Hit跟踪。 右键–>Hit跟踪–>标记模块不跟踪

    在Hit跟踪的基础上记录地址、寄存器的内容、消息以及已知的操作数。您还可以使用Run跟踪来了解运行的简单统计

    开始RUN跟踪 菜单–>调试–>开始或清除RUN跟踪
    查看RUN跟踪记录 (…按钮)

    在选择的代码上强迫设置Run跟踪和Hit跟踪。 右键–>添加选择部分
    在当前函数过程上强迫设置Run跟踪和Hit跟踪。 右键–>添加函数过程

    模糊命令

    在搜索命令时，您可以使用模糊汇编指令。可以使用下面的关键字：

    关键字 匹配
    R8 任何8位寄存器(AL,BL, CL, DL, AH, BH, CH, DH)
    R16 任任何16位寄存器(AX, BX, CX, DX, SP, BP, SI, DI)
    R32 任任何32位寄存器 (EAX, EBX, ECX, EDX, ESP, EBP, ESI, EDI)
    RA & RB 像R32一样，它们可以替代任何32位通用寄存器，但他们在整个序列中含义的总是不变的
    FPU 任任何FPU寄存器(ST0..ST7)
    MMX 任任何MMX寄存器(MM0..MM7)
    CRX 任任何控制寄存器(CR0..CR7)
    DRX 任任何调试寄存器r (DR0..DR7)
    CONST 任任何常量
    ANY 跳过n个命令
    OFFSET 与CONST相同
    JCC 任何条件跳转(JE, JC, JNGE…)
    SETCC 任何条件设置字节(SETE, SETC, SETNGE… )
    CMOVCC 任何条件移动 (CMOVE, CMOVC, CMOVNGE…)

    例子
    call CONST 可以匹配 call test.00401020
    LEA RA,[4 * RA + RA] 匹配到LEA EAX,[4 * EAX + EAX]
    PUSH EBP;ANY 2;MOV EBP,ESP 函数过程的头部可以有两条任意的命令
